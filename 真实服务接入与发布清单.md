# BetweenUs 真实服务接入与发布清单

这份清单只保留正式上线需要的动作，不再包含旧的支付宝/微信支付方案。

## 一、先判断能不能继续做

这两个前置条件没有通过，就不要继续堆代码：

1. `App 备案` 是否能用你当前主体通过。
2. `阿里云短信模板/签名审核` 是否能用你当前主体通过。

## 二、外部资源清单

### 账号与提审

1. Apple Developer 账号。
2. App Store Connect 应用条目。
3. Sign in with Apple 配置。
4. Apple IAP 商品：`betweenus.payg.1`、`betweenus.payg.2`、`betweenus.payg.3`。

### 中国大陆基础设施

1. 火山引擎云主机或等价大陆计算资源。
2. PostgreSQL。
3. Redis。
4. 火山 TOS 存储桶。
5. 已备案域名与 HTTPS 证书。

### AI 与短信

1. 火山录音识别 App ID / Access Token。
2. DeepSeek API Key。
3. 阿里云短信 AccessKey。
4. 阿里云短信签名、模板、方案名（如有）。

## 三、后端必须填的环境变量

以 `backend/.env.example` 为准，重点是这些：

1. `ENV=prod`
2. `DATABASE_URL=postgresql://...`
3. `JWT_SECRET_KEY=<强随机密钥>`
4. `ASR_PROVIDER=volc_recording_bigmodel`
5. `ASR_VOLC_UPLOAD_PROVIDER=volc_tos`
6. `VOLC_TOS_ENDPOINT / REGION / BUCKET / ACCESS_KEY_ID / ACCESS_KEY_SECRET`
7. `APPLE_CLIENT_ID / APPLE_TEAM_ID / APPLE_KEY_ID / APPLE_PRIVATE_KEY`
8. `SMS_PROVIDER=aliyun`
9. `SMS_ALIYUN_ACCESS_KEY_ID / SMS_ALIYUN_ACCESS_KEY_SECRET`
10. `SMS_ALIYUN_SIGN_NAME / SMS_ALIYUN_TEMPLATE_CODE`
11. `APPLE_IAP_BUNDLE_ID / APPLE_IAP_ENVIRONMENT / APPLE_IAP_ROOT_CA_PATHS`

## 四、你需要在 App Store Connect 做的事

1. 开启 `Sign in with Apple`。
2. 创建 3 个消耗型商品：
   - `betweenus.payg.1`
   - `betweenus.payg.2`
   - `betweenus.payg.3`
3. 配置 `App Store Server Notifications` 地址：
   - `POST /v1/billing/app-store-notifications`
4. 填好中国大陆上架需要的 `ICP Filing Number`。
5. 填好 `App Privacy` 数据项。

1. 当前预览地址：`https://betweenus-site.vercel.app/`
2. 正式官网：`https://betweenus.app/`
3. 正式支持页：`https://betweenus.app/support/`

7. GitHub 仓库：`https://github.com/qinyuhao84-ship-it/BetweenUs`

   - `betweenus.app`
   - `www.betweenus.app`

## 四点六、后台部署入口

1. 仓库根目录已经新增 `render.yaml`，可直接作为预览环境拉起：
   - `betweenus-api`
   - `betweenus-worker`
   - `betweenus-db`
   - `betweenus-redis`
2. 中国大陆正式环境使用仓库中的单 ECS 部署骨架：
   - `deploy/china/docker-compose.yml`
   - `deploy/china/Caddyfile`
   - `deploy/china/.env.example`
   - `deploy/china/release.sh`
3. 中国大陆正式 DNS 目标应当是：
   - `betweenus.app -> 中国大陆 ECS 公网 IP`
   - `www.betweenus.app -> 中国大陆 ECS 公网 IP`
   - `api.betweenus.app -> 中国大陆 ECS 公网 IP`
4. 后端镜像文件已准备好：
   - `backend/Dockerfile`
   - `backend/docker/start-web.sh`
   - `backend/docker/start-worker.sh`
5. Apple IAP 验签根证书已放入仓库：
   - `backend/certs/apple/AppleRootCA-G2.cer`
   - `backend/certs/apple/AppleRootCA-G3.cer`

## 五、发布前验收顺序

1. `Apple 登录` 能直接进入主流程。
2. `手机号验证码` 能真实收到。
3. `录音 -> 转写 -> 报告` 全链路成功。
4. `Apple IAP` 购买一次，只加一次额度。
5. `退款/撤销` 能回收额度。
6. `删除账号` 后，历史、额度、资料都消失。
7. Release 构建没有 ATS 放开项，API 和法律文档链接都是 HTTPS。
8. `healthz` 和 `readyz` 都正常。

## 六、上线前最后一轮人工检查

1. App 图标、截图、描述、关键词、支持 URL、营销 URL。
2. 隐私政策、用户协议、用户隐私选择页面已经挂到正式域名。
3. 联系邮箱和反馈入口可用。
4. App Review 备注里说明：
   - 审核员可直接用 Sign in with Apple
   - 中国大陆手机号验证码只用于正式用户
   - 支付为 Apple IAP 次数包
